# Shenzhen Metro Accessibility Analyzer (SZ-Metro-Metrics)

## Project Overview

A data engineering and analysis pipeline to calculate, aggregate, and visualize travel time complexity between Shenzhen Metro stations and administrative districts.

## Tech Stack & Architecture

- **Data Collection**: `Playwright` or `httpx` (Asynchronous scraping)
- **Data Processing**: `Pandas`, `Numpy`
- **Graph Theory**: `NetworkX` (For All-Pairs Shortest Path calculation)
- **Visualization**: `Matplotlib`, `Seaborn`, `Geopandas` (Optional for map overlay)
- **Coding Style**: PEP 8, strict type hinting, Google-style docstrings.

## Development Standards

- **Modular Design**: Separate logic into `scraper`, `engine`, `aggregator`, and `viz`.
- **Robustness**:
  - All network requests must implement exponential backoff retry.
  - Data schemas must be validated (use `Pydantic` models if complex).
- **Documentation**: Every function must include Type Hints and a short description.
- **Error Handling**: Custom exception classes for scraping failures or graph disconnectedness.

## System Workflow

1. **Source**: Extract station data (Name, Line, Transfers) and segment duration from API/Web.
2. **Graph Construction**: Stations as nodes, travel times as weighted edges, transfer times as penalty weights.
3. **Computation**: Use Dijkstra/Floyd-Warshall for the travel time matrix.
4. **Aggregation**: Group by `District` and calculate Mean/Median accessibility metrics.
5. **Categorization**: Classify "Complexity" (e.g., Simple, Moderate, Complex, Remote) based on quantile thresholds.

## Extensibility for Future Projects

- All data loaders should support `CSV` and `JSON` backends.
- Graph engine must accept arbitrary weighted edges to support Bus or High-speed rail in future versions.
